# Human Compatibility OS — возможности сервиса

Операционная HR-платформа в философии **Fabrika Sredy** («Среда важнее героизма»).
Сервис помогает увидеть и снизить **давление среды** (выгорание, перегрузку,
коммуникационный шум, нестабильность лидерства) и оценить **совместимость
человека со средой** — но никогда не оценивает «ценность человека» и не принимает
кадровых решений за людей.

> Все выводы — объяснимые сигналы для **проверки человеком**. Светофор-индикаторы
> не являются основанием для увольнения, отказа в найме или иного кадрового действия.

---

## Роли и доступ (RBAC)

| Роль | Что может |
|------|-----------|
| **Employee** (сотрудник) | Проходить опросник, видеть свой результат и историю, читать review о себе, экспортировать свои данные |
| **Team Lead** (руководитель) | Всё выше + дашборд, рекалибровка, calibration review, подбор — **только по своей команде** |
| **HR** | То же по всем командам |
| **Founder/Admin** | Полный доступ + аудит-журнал |
| **Ethics Reviewer** | Просмотр для надзора + аудит-журнал |

Аутентификация — JWT (access + refresh). Сбор операционных данных невозможен без
**явного согласия** сотрудника.

## Что умеет сервис

### 1. Опросник среды и оценка выгорания
- Опросник из **15 вопросов** (шкала 1–5) о давлении, восстановлении, коммуникации,
  отвлечениях, лидерстве.
- Движок считает **Burnout Pressure** по взвешенной формуле из 5 компонентов
  (давление аврала, дефицит восстановления, коммуникационная перегрузка, плотность
  отвлечений, нестабильность лидерства).
- **Объяснимый результат**: не только итоговый балл и уровень риска (низкий/средний/
  высокий), но и разбор по каждому компоненту — видно, что именно тянет вверх.
- История прохождений по сотруднику.

### 2. Дашборд среды команды
- Агрегаты по **4 блокам**: Давление выгорания, Устойчивость восстановления,
  Коммуникационная энтропия, Устойчивость лидерства.
- По каждому блоку — средний уровень и **распределение людей** по зонам риска.
- **Анонимизация**: если в выборке меньше 3 человек с данными, агрегаты
  скрываются — команда не может превратиться в профиль одного человека.
- Метрики среды (нагрузка встречами и т.п.) — сводка count / среднее / мин / макс.

### 3. Рекалибровка (динамика во времени)
- Циклы наблюдения: **базовая точка → 30 дней → 90 дней → ретроспектива**.
- **Тренд относительно базовой точки**: улучшение / ухудшение / без изменений.
- **Рекомендации по развитию среды** — привязаны к доминирующему фактору давления
  (например, «защитить время восстановления» или «сократить число каналов/встреч»).
  Носят развивающий характер, не являются предписанием.

### 4. Calibration Review (проверка человеком)
- HR / руководитель фиксируют **человеческую оценку риска** с обязательным
  указанием источника данных, рекомендацией и действиями.
- **Прозрачность**: сотрудник всегда может прочитать review о себе.
- Каждая запись и просмотр — в аудит-журнале.

### 5. Подбор и совместимость (Compatibility Hiring)
- **Quick Screen** — быстрая оценка по 4 сигналам (объяснимость, устойчивость к
  обратной связи, отношение к хаосу, ответственность) для первичного скрининга.
- **Full Calibration** — глубокая оценка по 10 сигналам + измерения совместимости
  (долгосрочная совместимость, тактическая ценность, стоимость хаоса, влияние на
  команду, потенциал роста) для ключевых и лидерских ролей.
- Встроенный **справочник**: по каждому сигналу — поведенческий индикатор, вопрос
  для интервью, фокус наблюдения и легенда уровней риска.
- Система **подсказывает** общий риск (зелёный/жёлтый/красный), но решение —
  всегда за человеком. **Никакого автоматического отказа.**
- **План развития** кандидата (зона риска → поддержка → прогресс).

### 6. Пилотные метрики (эффект за 90 дней)
- Ключевая метрика пилота: **снижение давления аврала ≥ 20% за 90 дней**.
- Плюс динамика всех 4 блоков среды (базовая точка → 90 дней) с пометкой улучшения.

### 7. Онбординг новых сотрудников
- **Трение интеграции**: сравнение давления среды у новичков (в команде < 90 дней)
  с устоявшейся командой — видно, тяжелее ли адаптация у новых людей.
- Счётчик новичков в зоне высокого риска.

### 8. Этика, комплаенс, аудит, экспорт
- **Машиночитаемая доктрина** (`/compliance/policy`): подтверждает «нет авто-решений»,
  «требуется согласие», «обязательна проверка человеком».
- **Аудит-журнал** (для admin / ethics reviewer): кто что сделал и что просматривал —
  подотчётность вместо скрытого профайлинга.
- **Экспорт для проверки человеком**: единый объяснимый пакет по сотруднику
  (профиль + история опросников + рекалибровка + reviews) одним файлом.

## Принципы, зашитые в продукт

- **Explainability-first** — каждый вывод объясним и проверяем.
- **Никаких автоматических решений** — индикаторы только информируют человека.
- **Обязательная проверка человеком** для всех выводов.
- **Явное согласие** и прозрачность сбора данных.
- **Анти-слежка** — подавление малых выборок, аудит доступа, оценка среды, а не
  слежка за человеком.

## Чего сервис принципиально НЕ делает

- Не увольняет, не отклоняет кандидатов и не понижает автоматически.
- Не ставит психологических диагнозов — это операционный сигнальный слой, не
  психометрия.
- Не строит скрытых профилей и не показывает индивидуальные данные под видом
  командной аналитики.

## Технические факты

- **Стек:** FastAPI + SQLAlchemy + PostgreSQL (backend), Next.js + Tailwind (frontend, UI на русском), Docker, Caddy (HTTPS), GitHub Actions CI.
- **Экраны:** опросник, дашборд, рекалибровка, calibration review, аудит, подбор.
- **Тестовый стенд:** развёртывается одним docker-compose (см. [DEPLOY.md](DEPLOY.md)).
- **Дорожная карта и статус:** [ROADMAP.md](ROADMAP.md); инструкция пилота: [PILOT.md](PILOT.md).
