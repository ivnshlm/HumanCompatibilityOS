# Роли и доступ — инструкция

Как работают роли в Human Compatibility OS: что доступно каждой роли, что можно
делать и как. Стенд: **https://2.56.212.169.sslip.io** (API/Swagger: `/api/docs`).

## Общие правила

- **Вход:** `/login` (email + пароль).
- **Регистрация** нового аккаунта — через Swagger `/api/docs` → `POST /auth/register`
  (создаётся роль «Сотрудник»; в веб-интерфейсе формы регистрации нет).
- **Роль назначает только администратор** на экране `/admin`. Роль **действует сразу**
  — перелогин не нужен (читается из БД на каждый запрос).
- В шапке видны только разделы, доступные вашей роли; остальные отвечают
  «доступ только для…».
- Все изменения ролей, просмотры и записи попадают в **аудит-журнал**.

## Матрица доступа (экран / действие → роли)

| Экран / действие | Сотрудник | Тимлид | HR | Этич. ревьюер | Админ |
|---|:--:|:--:|:--:|:--:|:--:|
| Опросник, свой результат и история | ✅ | ✅ | ✅ | ✅ | ✅ |
| Дашборд среды (`/dashboard`) | — | своя команда | ✅ | 👁 чтение | ✅ |
| Запись метрик среды | — | ✅ | ✅ | — | ✅ |
| Рекалибровка (`/recalibration`) | своя | своя команда | ✅ | ✅ | ✅ |
| Calibration Review (`/review`) | 👁 о себе | своя команда | ✅ | ✅ | ✅ |
| Подбор / совместимость (`/hiring`) | — | ✅ | ✅ | 👁 просмотр | ✅ |
| Аудит-журнал (`/audit`) | — | — | — | ✅ | ✅ |
| Управление пользователями (`/admin`) | — | — | — | — | ✅ |
| Экспорт по сотруднику | свой | своя команда | ✅ | ✅ | ✅ |

«Своя команда» = тимлид ограничен своим `team_id`. HR/админ — без ограничения по команде.

## По ролям

### 👤 Сотрудник (employee)
**Доступно:** опросник, свой результат с разбором и интерпретацией, своя история,
чтение review о себе, экспорт своих данных.
**Как:**
1. `/login` → войти.
2. **«Пройти опросник»** → выбрать длину сессии (короткая / базовая / глубокая) →
   отметить **согласие** → ответить по шкале 1–5 → «Отправить».
3. Сразу виден результат: балл, уровень риска, разбор по компонентам, интерпретация.
4. Своя история прохождений — в профиле / по своему `id`.

### 👥 Тимлид (team_lead)
**Доступно:** всё, что у сотрудника, **плюс по своей команде** — дашборд, запись
метрик среды, рекалибровка, calibration review, подбор, экспорт. Чужие команды — нет.
**Как:**
- **Дашборд:** шапка → «Дашборд» (показывает вашу команду по вашему `team_id`).
- **Review:** «Review» → выбрать сотрудника своей команды → заполнить (уровень риска,
  рекомендация, **источник данных**, действия) → сохранить.
- **Подбор:** «Подбор» → добавить кандидата → Quick Screen (4 сигнала) или
  Full Calibration (10 сигналов) → план развития.

### 🧑‍💼 HR
**Доступно:** то же, что у тимлида, но **по всем командам** (без ограничения `team_id`).
**Как:** аналогично тимлиду. На дашборде видите команду, привязанную к вашему профилю
(`team_id` задаётся админом в `/admin`); по API/экспорту доступны любые команды.

### 🛡️ Этич. ревьюер (ethics_reviewer)
**Доступно:** надзор — **чтение** дашборда, рекалибровки, подбора, экспорта **и
аудит-журнал**. Запись метрик среды — нет. Роль ориентирована на контроль принципов
(«нет авто-решений», приватность, анонимизация).
**Как:**
- **Аудит:** шапка → «Аудит» → таблица всех действий и просмотров (кто что делал).
- Остальные разделы открываются в режиме просмотра.

### ⚙️ Администратор (admin)
**Доступно:** всё вышеперечисленное **+ аудит + управление пользователями**.
**Как управлять пользователями:**
1. `/login` под админом → шапка **«Админ»** (`/admin`).
2. У каждого пользователя: выпадающий список **роли**, поле **`team_id`** (UUID
   команды), кнопка **Активен / Выключен**.
3. Изменения применяются сразу и пишутся в аудит.
4. Защита: нельзя разжаловать/отключить последнего активного админа и нельзя
   отключить собственный аккаунт.

## Как выполнить ключевые действия (пошагово)

**Завести нового сотрудника и дать роль**
1. Swagger `/api/docs` → `POST /auth/register` → `{email, password, full_name}` →
   создаётся employee.
2. Админ → `/admin` → выбрать роль и при необходимости вписать `team_id` → готово.

**Первый админ на чистой системе** — прописать e-mail(ы) в `INITIAL_ADMIN_EMAILS`
(через запятую) в `.env.prod` на сервере; этот e-mail при регистрации становится
админом автоматически.

**Собрать команду на дашборде** — всем участникам проставить **одинаковый `team_id`**
(в `/admin`); после прохождения опросника дашборд по этому `team_id` наполнится.
Нужно **≥ 3 человек** с данными, иначе агрегаты скрыты анонимизацией.

**Пилотный цикл (тренд −20% за 90 дней)** — сотрудник проходит опросник → событие
рекалибровки `baseline`; через 30/90 дней повторный опросник и события
`day_30` / `day_90`; на дашборде появляются пилот-метрика и тренд.

**Экспорт для проверки человеком** — на `/review` выбрать сотрудника → «Экспорт для
проверки» (скачивает JSON: профиль + опросники + рекалибровка + reviews).

## Технические детали прав (для справки)

- Регистрация всегда создаёт `employee` (роль клиент задать не может — защита от
  самоповышения). Исключение — bootstrap по `INITIAL_ADMIN_EMAILS`.
- Роли/команду/активность меняет только админ: `PATCH /admin/users/{id}`.
- RBAC по эндпоинтам:
  - Дашборд / рекалибровка / онбординг / пилот-метрика / директория / экспорт —
    `hr, team_lead, admin, ethics_reviewer` (тимлид — своя команда).
  - Запись метрик среды — `hr, team_lead, admin`.
  - Подбор: создание — `hr, team_lead, admin`; просмотр — плюс `ethics_reviewer`.
  - Calibration review: создание/просмотр — `hr, team_lead, admin, ethics_reviewer`
    (тимлид — своя команда); сотрудник читает review о себе.
  - Аудит-журнал — `admin, ethics_reviewer`.
  - Управление пользователями — `admin`.
- Текущий рабочий админ для проверки: `admin@env-fabric.ru` (рекомендуется завести
  свой админ-аккаунт и сменить демо-пароль).
